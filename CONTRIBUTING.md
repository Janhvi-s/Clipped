# Contributing to Clipped

Thanks for your interest in contributing! Clipped is a personal learning tool built in the open, and contributions of all kinds are welcome — bug fixes, features, documentation improvements, or just ideas.

## Getting started locally

1. Fork the repo and clone your fork
2. Follow the setup steps in [README.md](README.md) to get the app running
3. Create a new branch for your change:
   ```bash
   git checkout -b your-feature-name
   ```
4. Make your changes, test them locally
5. Push to your fork and open a Pull Request

## Submitting a PR

- Keep PRs focused — one feature or fix per PR
- Write a clear description of what you changed and why
- If you're fixing a bug, mention the steps to reproduce it
- If you're adding a feature, briefly explain the use case

## Good first issues

Not sure where to start? Look for issues tagged `good first issue` or `help wanted`. Small improvements like better error messages, UI tweaks, or documentation fixes are always appreciated.

## Questions

Open an issue if you're unsure about something before spending time building it. Happy to discuss ideas early.

First-time contributors are very welcome here.
